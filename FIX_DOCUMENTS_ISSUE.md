# Исправление проблемы с загрузкой документов

## ❌ Проблема
HR загрузил документы для сотрудника (Купешев Мадияр), но они не отображались в UI.

## 🔍 Причина
В таблице `documents` отсутствовала колонка `thumbnail_url`, которую бэкенд пытался записать при загрузке файлов.

## ✅ Решение
Добавлена миграция `004_add_document_columns.sql` которая добавляет:
- `thumbnail_url TEXT` - для миниатюр изображений
- `egov_operation_id TEXT` - для SIGEX интеграции
- `egov_status TEXT` - для статуса SIGEX
- `signers JSONB` - для информации о подписантах

## 🔧 Применение
Миграция уже применена к production базе данных.

Для локальной разработки:
```bash
cd database
psql $DATABASE_URL -f migrations/004_add_document_columns.sql
```

## 📝 Проверка
1. Открыть профиль сотрудника: `/hr/employees/:id`
2. Перейти во вкладку "Документы"
3. Загрузить новый документ (Уд. личности, Фото и т.д.)
4. Документ должен отобразиться с превью
