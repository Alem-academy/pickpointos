# Техническая документация: PickPoint Operations (AlemLab)

Данный документ описывает общую архитектуру, стек технологий, структуру базы данных и основные рабочие потоки проекта.

## 1. Обзор архитектуры

Приложение построено по клиент-серверной архитектуре и разделено на несколько директорий/сервисов:
- **Frontend (`/src`)**: SPA-приложение на React, предоставляющее пользовательские интерфейсы для сотрудников ПВЗ (Пунктов Выдачи Заказов), HR-специалистов, региональных менеджеров (RF), финансистов и администраторов.
- **Backend API (`/parser`)**: Основной Node.js сервер, обслуживающий REST API для фронтенда и взаимодействующий с PostgreSQL.
- **Sigex Gateway (`/sigex-gateway`)**: Отдельный микросервис Node.js, созданный для интеграции с сервисом цифровых подписей SIGEX (eGov QR).
- **База Данных (`/database`)**: Скрипты инициализации (migration, seed, schema) для СУБД PostgreSQL.

## 3. База Баных (PostgreSQL)

Схема базы данных построена на использовании `UUID` и связей для обеспечения целостности.

### Основные таблицы:
1. `employees` (Сотрудники)
   *   Хранит данные сотрудников: ИИН, ФИО, электронную почту, телефон.
   *   Ролевая модель `role`: `admin`, `hr`, `rf` (Regional Facilitator), `employee`.
   *   Статусы `status`: `new`, `review`, `revision`, `signing`, `active`, `fired`.
   *   Доп. поля: `rejection_reason` (причина возврата HR-ом на доработку), `emergency_contacts` (JSONB с контактами родственников).
2. `pvz_points` (Пункты выдачи заказов)
   *   Информация о точках (имя, адрес, площадь, ID в системе Wildberries).
3. `shifts` (Смены)
   *   График рабочего времени сотрудников на ПВЗ (плановые и фактические часы работы).
4. `documents` (Документы)
   *   Трудовые договоры, сканы УДЛ сотрудников (через S3 URL) и статусы подписания (`draft`, `signed`, `rejected`).
5. `expenses_requests` & `financial_transactions` & `fact_payroll` (Финансы)
   *   Учет финансовых транзакций: аренда, коммуналка, ЗП, штрафы, доходная часть (отчёты WB).
   *   Связываются с `pvz_points`.

## 4. Ключевые бизнес-процессы (Flows)

### 4.1. HR Отношения и Onboarding (Найм)
Система покрывает процесс "от сбора анкеты до подписания договора":
1.  **Сбор данных**: HR создает анкету (или кандидат заполняет её по ссылке-приглашению). Проверяются ИИН, Адрес, IBAN. Указываются контакты родственников.
2.  **Загрузка документов**: Кандидат прикрепляет сканы удостоверения личности и справки.
3.  **Проверка HR**: HR может вернуть документы на доработку с указанием причины (статус `revision`), либо подтвердить их (`review` -> `signing`).
4.  **Генерация договора**: Если всё верно, система генерирует трудовой договор в PDF.

### 4.2. Интеграция с SIGEX (Подписание eGov QR)
Процесс подписания трудового договора сотрудником через eGov Mobile:
1.  Фронтенд запрашивает у `sigex-gateway` регистрацию документа для подписания.
2.  SIGEX генерирует QR-код, который фронтенд отображает пользователю.
3.  Пользователь сканирует QR через приложение *eGov Mobile* и подписывает документ (ЭЦП).
4.  Gateway ловит коллбэк/статус, извлекает **ИИН** из CMS-подписи, накладывает графический штамп "Подписано ЭЦП" на исходный PDF с помощью библиотеки `pdf-lib` и обновляет статус документа на подписанный в базе.
5.  Если ИИН из ключа не совпадает с ИИН из анкеты, система отмечает расхождение.

### 4.3. Управление ПВЗ (RF Dashboard)
Regional Facilitators следят за работой сетей пунктов выдачи:
*   Одобряют заявки на расходы (покупка оборудования, оплата аренды).
*   Просматривают PnL отчетность (Profit & Loss). Финансовые данные парсятся из предоставленных Excel/таблиц отчетом Wildberries с помощью `parser-service` и сохраняются в `fact_wb_revenues`.

## 5. Дополнительная информация

### Основные команды для разработки:
* Запуск фронтенда: `npm run dev` (корневая папка)
* Запуск базы данных и сидов (нужно настроить `.env`): `npm run db:seed`
* Запуск локального API-бекэнда: `npm run start` (или `node parser/src/index.js`)
* Запуск Sigex Gateway: `cd sigex-gateway && npm run dev`

### Настройка переменных окружения (Environment Variables)
Для работы необходимо настроить `.env` файлы в корне и в модулях бэкенда.
Ключевые переменные:
*   `DATABASE_URL` (подключение к PostgreSQL)
*   `JWT_SECRET` (ключ для токенов авторизации)
*   `AWS_ACCESS_KEY_ID` / `AWS_SECRET_ACCESS_KEY` / `AWS_BUCKET_NAME` (для S3 хранилища)
*   Адреса SIGEX API для gateway.
