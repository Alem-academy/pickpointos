# Документация по HTML-шаблонам документов для ЭЦП

## Общая информация

Все шаблоны находятся в директории `document-templates/` и предназначены для генерации документов на подпись (ЭЦП). Шаблоны используют синтаксис переменных `{{variableName}}` для подстановки данных из системы.

## Соглашения по именованию

### Формат переменных
- Используется синтаксис Mustache: `{{variableName}}`
- Имена переменных в camelCase
- Только латинские символы

### Языковые суффиксы (КРИТИЧНО)
Каждая переменная, содержащая языко-зависимый текст, должна иметь суффикс языка:
- `*Kz` — значение на **казахском** языке (қазақша)
- `*Ru` — значение на **русском** языке
- Без суффикса — универсальное значение (числа, ИИН, номера документов)

**Правило:** в казахском блоке шаблона используются ТОЛЬКО `*Kz` переменные. В русском блоке — ТОЛЬКО `*Ru`.

### Падежи в переменных
- `*Im` — именительный падеж (кто? что?) — используется редко, обычно без суффикса
- `*Rod` — родительный падеж (кого? чего?)
- `*Dat` — дательный падеж (кому? чему?)
- `*Vin` — винительный падеж (кого? что?) — для глаголов «принять», «уволить», «назначить»
- `*Tvor` — творительный падеж (кем? чем?)
- Без суффикса — именительный падеж (кто? что?)

### Примеры именования
```
{{dateMonthKz}}      — месяц на казахском: "сәуір"
{{dateMonthRu}}      — месяц на русском: "апреля"
{{employeeAddressKz}} — адрес на казахском: "Алматы қаласы, Абай көшесі, 90 үй"
{{employeeAddressRu}} — адрес на русском: "г. Алматы, ул. Абая, д. 90"
{{employeeFullNameVin}} — ФИО в винительном падеже: "Локутневскую Дарью Максимовну"
{{idCardIssuerKz}}   — орган выдачи на казахском: "ҚР ІІМ"
{{idCardIssuerRu}}   — орган выдачи на русском: "МВД РК"
```

### Даты
Даты разбиты на составляющие для гибкости:
- `{{dateDay}}` — день числом (универсально)
- `{{dateMonthKz}}` — месяц на казахском: `қаңтар`, `ақпан`, `наурыз`...
- `{{dateMonthRu}}` — месяц на русском: `января`, `февраля`, `марта`...
- `{{dateYear}}` — год
- `{{currentDate}}` — полная дата в свободном формате

### Перевод адресных элементов
При подготовке данных для шаблонов адреса должны быть переведены:

| Русский | Казахский |
|---------|-----------|
| г. / город | қаласы |
| ул. / улица | көше |
| д. / дом | үй |
| кв. / квартира | пәтер |
| обл. / область | облысы |
| р/н / район | ауданы |
| учетный квартал | есептік квартал |
| РК / Республика Казахстан | ҚР / Қазақстан Республикасы |
| МВД РК | ҚР ІІМ |

## Стили и верстка

### Базовые принципы
- Каждый шаблон самодостаточен (встроенные `<style>`)
- Размер страницы: A4 (`210mm x 297mm`)
- Шрифт: Times New Roman, 12-14pt
- Отступы: 15-20mm
- Черный цвет текста (`#000`)

### Печать
Все шаблоны содержат CSS-правило:
```css
@page { size: A4; margin: 20mm; }
```

### Генерация PDF
Для генерации PDF рекомендуется использовать Puppeteer, Playwright или аналогичный инструмент с поддержкой `@page`:
```javascript
const puppeteer = require('puppeteer');
const html = renderTemplate(templateString, data);
const browser = await puppeteer.launch();
const page = await browser.newPage();
await page.setContent(html, { waitUntil: 'networkidle0' });
await page.pdf({ format: 'A4', printBackground: true });
```

## Падежи в кадровых документах

### Когда какой падеж

| Действие | Падеж | Пример |
|----------|-------|--------|
| Принять кого? | **Винительный** (Vin) | Принять **Локутневскую Дарью Максимовну** |
| Уволить кого? | **Винительный** (Vin) | Уволить **Иванова Ивана Ивановича** |
| Предоставить кому? | **Дательный** (Dat) | Предоставить **Ивановой Марии Петровне** |
| Приказ о ком? | **Родительный** (Rod) | Приказ о предоставлении отпуска **Ивановой М.П.** |
| Заявление от кого? | **Родительный** (Rod) | от менеджера **Ивановой М.П.** |
| Назначить кем? | **Творительный** (Tvor) | Назначить **Ивановой М.П.** ответственной |

### Рекомендация по реализации склонения
Для автоматического склонения ФИО рекомендуется:
1. Использовать библиотеку `petrovich` (JavaScript/Node.js) для русских ФИО
2. Для казахских ФИО использовать правила казахской морфологии (обычно не требует склонения)
3. Или хранить предварительно просклоненные ФИО в БД для каждого сотрудника

## Рекомендации по интеграции

### 1. Рендеринг шаблонов
Рекомендуется использовать легковесный шаблонизатор, например Handlebars или Mustache:

```javascript
import Handlebars from 'handlebars';

const template = Handlebars.compile(templateSource);
const html = template(data);
```

### 2. Подготовка данных
Перед рендерингом необходимо:
1. Склонять ФИО и должности в нужные падежи
2. Преобразовать даты в текстовый формат (число + месяц прописью)
3. Числа в текст (дни прописью для приказов)
4. Перевести адреса и названия органов на нужный язык

### 3. Валидация языка
```javascript
function validateLanguage(data, templateType) {
  // Проверка: в русских переменных нет казахских символов
  const kazakhChars = /[әіңғүұқөһӘІҢҒҮҰҚӨҢ]/;
  
  if (kazakhChars.test(data.dateMonthRu)) {
    throw new Error('dateMonthRu содержит казахские символы!');
  }
  
  if (kazakhChars.test(data.employeeAddressRu)) {
    throw new Error('employeeAddressRu содержит казахские символы!');
  }
}
```

### 4. Генерация PDF
```javascript
import puppeteer from 'puppeteer';

async function generatePDF(html, outputPath) {
  const browser = await puppeteer.launch();
  const page = await browser.newPage();
  await page.setContent(html, { waitUntil: 'networkidle0' });
  await page.pdf({
    path: outputPath,
    format: 'A4',
    printBackground: true,
    margin: { top: '20mm', right: '20mm', bottom: '20mm', left: '20mm' }
  });
  await browser.close();
}
```

### 5. Подписание документов
После генерации PDF:
1. Отправить PDF на подпись через Sigex Gateway
2. Сохранить подписанную версию в хранилище документов
3. Обновить статус документа в БД

## Важные замечания

1. **Двуязычность**: Приказы и ТД содержат казахский и русский языки. При генерации оба блока должны быть заполнены корректными данными на соответствующем языке.
2. **Подписи**: В HTML-шаблонах подписи отображаются как линии. После подписания ЭЦП линии заменяются на изображения подписей или штампы.
3. **Пустые значения**: Если переменная может быть пустой, рекомендуется скрывать соответствующий блок или выводить прочерк.
4. **Валидация**: Перед генерацией необходимо проверить наличие всех обязательных переменных и корректность языка.
